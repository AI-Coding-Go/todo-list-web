# 获取提醒设置

## 接口信息

- **URL**: `/api/settings/reminder`
- **方法**: `GET`
- **描述**: 获取全局提醒开关状态

## 请求参数

无

## 请求示例

```bash
curl http://localhost:8080/api/settings/reminder
```

```javascript
fetch('http://localhost:8080/api/settings/reminder')
  .then(response => response.json())
  .then(data => console.log(data));
```

## 响应示例

### 成功响应

```json
{
  "code": 200,
  "message": "操作成功",
  "data": true
}
```

## 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| data | Boolean | 提醒开关状态：true-开启，false-关闭。默认值为true |

## 业务规则

1. 默认值为true（开启）
2. 如果从未设置过，返回true
3. 此设置影响所有任务的提醒功能

## 注意事项

1. 此设置是全局设置，影响所有任务
2. 如果设置为false，`/api/reminders`接口将返回空数组
3. 前端可以在设置页面显示此开关状态

## 使用场景

- 在设置页面显示提醒开关状态
- 根据开关状态决定是否启动提醒轮询
