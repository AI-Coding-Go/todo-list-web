# 获取任务详情

## 接口信息

- **URL**: `/api/tasks/{id}`
- **方法**: `GET`
- **描述**: 根据任务ID获取任务详细信息

## 请求参数

### Path参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Long | 是 | 任务ID |

## 请求示例

```bash
curl http://localhost:8080/api/tasks/1
```

```javascript
fetch('http://localhost:8080/api/tasks/1')
  .then(response => response.json())
  .then(data => console.log(data));
```

## 响应示例

### 成功响应

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": 1,
    "title": "完成项目文档",
    "description": "编写API文档和用户手册",
    "status": 0,
    "statusDesc": "未完成",
    "priority": 1,
    "priorityDesc": "高",
    "dueTime": "2024-12-31T23:59:00",
    "overdue": true,
    "createTime": "2025-11-14T12:45:38",
    "updateTime": "2025-11-14T12:45:38",
    "finishTime": null
  }
}
```

### 失败响应（任务不存在）

```json
{
  "code": 500,
  "message": "任务不存在"
}
```

## 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 任务ID |
| title | String | 任务标题 |
| description | String | 任务描述，可能为null |
| status | Integer | 任务状态：0-未完成，1-已完成 |
| statusDesc | String | 状态描述："未完成" 或 "已完成" |
| priority | Integer | 优先级：1-高，2-中，3-低 |
| priorityDesc | String | 优先级描述："高"、"中" 或 "低" |
| dueTime | String | 截止时间，ISO 8601格式，可能为null |
| overdue | Boolean | 是否逾期。仅当status=0（未完成）且dueTime不为null且已过期时为true |
| createTime | String | 创建时间，ISO 8601格式 |
| updateTime | String | 最后更新时间，ISO 8601格式 |
| finishTime | String | 完成时间，ISO 8601格式。仅当status=1（已完成）时有值，否则为null |

## 业务规则

1. 任务ID必须存在，否则返回错误
2. 返回完整的任务信息，包括所有字段
3. 已完成的任务，finishTime字段有值
4. 未完成的任务，finishTime字段为null

## 注意事项

1. 任务ID必须是有效的数字
2. 如果任务不存在，返回错误信息
3. 时间字段使用ISO 8601格式：`YYYY-MM-DDTHH:mm:ss`
4. overdue字段仅在任务未完成且有截止时间且已过期时为true
