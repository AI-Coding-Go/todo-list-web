# 个人待办清单功能需求

## I. 项目概况

### 项目目标
开发一款轻量型个人待办清单小功能，支持待办任务的新增，编辑，删除，标记完成/未完成，列表查看，图表，待办提醒的核心功能。适配 web ios android 三个终端。

### 产品概述

#### 1. 产品定位
一款跨终端（Web/iOS/Android）轻量型个人待办清单工具，聚焦「任务管理效率」与「极简交互体验」，通过核心功能闭环满足用户日常待办记录、进度追踪、时间提醒需求，无冗余功能，专注提升个人事务管理效率。

#### 2. 核心价值
- 极简操作
- 全终端同步
- 智能提醒
- 数据洞察

### 核心功能需求

#### 任务基础管理模块

**任务新增**
- **必填信息：**
  - 任务标题（限 50 字以内，支持中英文/数字/符号）
  - 任务状态（默认「未完成」，不可手动修改，通过「标记完成」功能变更）
- **可选信息：**
  - 任务描述（限 500 字以内，支持换行，用于补充任务细节）
  - 截止时间（精确到「年-月-日 时：分」，未设置则为「无截止时间」）
  - 优先级（高/中/低三级，默认「中」，通过颜色区分：高=红色、中=灰色、低=绿色）
- **交互规则：**
  - 点击「+」按钮唤起新增表单，表单支持「取消」（丢弃输入）和「保存」（提交新增）
  - 标题为空时点击「保存」，提示「请输入任务标题」
  - 保存成功后，任务实时同步至列表顶部（按「新增时间倒序」排序）

**任务编辑**
- **可编辑内容：** 标题、描述、截止时间、优先级（同新增字段）
- **触发方式：**
  - 列表页点击任务右侧「编辑」按钮（图标：铅笔）
  - 点击任务进入详情页后，点击「编辑」按钮
- **交互规则：**
  - 编辑表单预填当前任务信息，修改后点击「保存」生效
  - 编辑过程中支持「取消」（保留原数据）
  - 编辑后任务在列表中的位置不变（仅更新内容）

**任务删除**
- **触发方式：**
  - 列表页左滑任务（移动端）或 hover 显示「删除」按钮（Web 端）
  - 详情页点击「删除」按钮
- **交互规则：**
  - 点击删除后弹出二次确认框：「确定删除该任务？删除后不可恢复」，支持「取消」和「确认删除」
  - 确认后任务立即从列表中移除，数据永久删除（不支持回收站）

**标记完成/未完成**
- **触发方式：**
  - 列表页点击任务左侧复选框（Web/iOS/Android 通用）
  - 详情页点击「标记完成」/「标记未完成」按钮（状态互斥）
- **视觉反馈：**
  - 标记完成后：标题显示删除线，文字颜色变浅（如灰色），复选框勾选
  - 标记未完成后：删除线和浅色效果取消，复选框未勾选

#### 任务列表查看模块

**列表展示**
- **默认视图：** 按「新增时间倒序」排列所有任务（最新任务在顶部），每行展示：
  - 复选框（标记完成/未完成）
  - 任务标题（含优先级颜色标签：高=红色圆点、中=灰色圆点、低=绿色圆点）
  - 截止时间（若设置，格式：「MM-DD HH:MM」，过期未完成显示红色并标「逾期」）
  - 操作按钮（编辑/删除，移动端左滑显示，Web 端 hover 显示）

**分组筛选：** 支持按「状态」筛选：
- 全部（默认）：展示所有任务
- 未完成：仅展示未完成任务（含逾期未完成）
- 已完成：仅展示已完成任务

**排序方式：** 支持切换排序规则（点击排序按钮切换）：
- 最新添加（默认）：按新增时间倒序
- 即将截止：按截止时间正序（无截止时间的任务排在最后）
- 优先级：高优先级→中→低（同优先级按新增时间倒序）

**任务详情页**
- **展示内容：**
  - 标题（大号字体，左侧带优先级颜色标签）
  - 状态标签（未完成/已完成，已完成显示「完成于：YYYY-MM-DD HH:MM」）
  - 截止时间（若设置，格式：「截止时间：YYYY-MM-DD HH:MM」，逾期未完成标红）
  - 任务描述（若有，换行展示）
  - 操作按钮：「编辑」「删除」「标记完成/未完成」（根据当前状态显示对应按钮）
- **跳转逻辑：** 列表页点击任务标题（非复选框区域）进入详情页，详情页支持「返回列表」

#### 任务数据图表模块

**统计维度**
- **完成率统计：**
  - 近 7 天任务完成率（已完成任务数/总任务数）
  - 按优先级统计完成率（高/中/低优先级分别计算）
- **任务数量趋势：**
  - 近 7 天每日新增任务数（折线图）
  - 近 7 天每日完成任务数（折线图，与新增趋势同图对比）
- **状态分布：**
  - 当前所有任务中「未完成」「已完成」的数量占比（饼图）

**图表展示**
- **入口：** 列表页底部点击「数据统计」按钮进入图表页
- **交互：**
  - 支持切换图表类型（点击标签切换「完成率」「趋势图」「状态分布」）
  - 图表数据实时更新（基于当前任务数据）
  - 无数据时显示「暂无统计数据，添加任务后即可查看」

#### 任务待办提醒模块

**提醒触发条件**
- 仅对「设置了截止时间且未完成」的任务触发提醒
- **提醒时间点：**
  - 截止时间前 30 分钟
  - 截止时间整点（若 30 分钟前已提醒，整点再次提醒）
  - 逾期后每 24 小时提醒 1 次（最多提醒 3 次，避免骚扰）

**多终端提醒方式**
- **Web 端：**
  - 浏览器桌面通知（需用户授权通知权限）
  - 页面顶部悬浮提醒条（含「查看任务」「稍后提醒」按钮）
- **iOS 端：**
  - 系统通知（横幅/锁屏通知，显示任务标题和截止时间）
  - 点击通知跳转至对应任务详情页
- **Android 端：**
  - 系统通知栏提醒（显示任务标题和截止时间）
  - 点击通知跳转至对应任务详情页

**提醒设置**
- 支持全局开启/关闭提醒（设置页开关：「允许任务提醒」，默认开启）
- 不支持单独设置某一任务的提醒方式（保持轻量，统一遵循全局设置）

## 补充点
- 本项目为单用户个人版，无需用户注册、登录、鉴权及多用户隔离。、
- 不做附件、标签、子任务、协作等扩展数据，仅实现文档要求功能。
- Web端提醒只用服务端定时器推送
- 所有后端API均按RESTful风格设计，返回统一JSON，排序/分页等API参数自行设计。
- 其他未明确写进需求文档的扩展功能、容错或高可用性不做实现。
- 后端推送提醒方案，采用“简单长轮询”方式，由前端定时请求接口查询未完成任务的提醒事件，后台负责计算并返回数据，不涉及 WebSocket。
- 需要完整接口文档（如 Swagger/OpenAPI）。
- 需要后端主要功能（service层、部分controller）对应的单元测试用例和必要的测试样例数据。
- 只需支持开发启动（本地 IDE 或命令行 run），不需打包Docker或脚本部署。
